# Rovo Description Generation Contract

This contract defines how plain-English catalog descriptions are generated by
Rovo from deterministic metadata evidence.

It implements:

- `docs/adr/ADR-017-Rovo-Assisted-Plain-English-Catalog-Descriptions.md`
- `docs/adr/ADR-015-Rovo-Optimized-AI-Retrieval-Artifacts.md`
- `docs/CONFLUENCE_HUMAN_LINEAGE_PAGE_CONTRACT.md`

## Purpose

Make new database onboarding sustainable by letting Rovo write direct support
language for strong and medium evidence while deterministic Node.js engines
continue to own metadata, lineage, page structure, validation, and publishing.

Codex must not be used as the LLM author for catalog descriptions in this
workflow.

## Required Evidence Packet

Every Rovo description request must be built from a bounded evidence packet.

```json
{
  "canonical_id": "",
  "page_type": "database | schema | object",
  "server": "",
  "database": "",
  "schema": "",
  "object": "",
  "object_type": "",
  "columns": [],
  "key_columns": [],
  "upstream": [],
  "downstream": [],
  "orchestrators": [],
  "reports": [],
  "packages": [],
  "pipelines": [],
  "procedures": [],
  "tags": [],
  "confidence": {
    "label": "strong | medium | weak",
    "reasons": [],
    "missing_facts": [],
    "caveats": []
  },
  "canonical_human_page": "",
  "rovo_context_page": "",
  "evidence_hash": "",
  "generated_at": ""
}
```

Do not include raw rows, sample values, secrets, credentials, or connection
strings.

## Rovo Output Shape

Rovo must return structured text that can be imported without hand editing:

```json
{
  "canonical_id": "",
  "purpose": "",
  "business_use": "",
  "support_notes": [],
  "lineage_summary": "",
  "confidence_used": "strong | medium",
  "unsupported_facts": [],
  "evidence_hash": ""
}
```

## Required Language Style

Use direct support language.

Good:

```text
Organization stores business structure and reference information used to
identify organizational entities across downstream reporting and support
processes.
```

Avoid weak page-about-page language:

```text
This page helps users understand the Organization database.
```

Use the standard footer on every human page:

```text
For questions or corrections, contact the Data Team.
```

## Auto-Publish Rules

Rovo output can be auto-published when:

- evidence confidence is `strong` or `medium`;
- returned `evidence_hash` matches the current packet;
- output contains the required fields;
- output does not invent unsupported owner, steward, SLA, lifecycle/status,
  freshness, certification, or source-system claims;
- output names actual metadata objects when explaining lineage or support
  checks;
- validation passes.

Weak evidence must use deterministic fallback text.

## Override Rules

Human-approved overrides win over Rovo output and deterministic fallback text.

Override files should be repo-backed and reviewed like source artifacts. Future
refreshes must preserve overrides unless a user explicitly removes or replaces
them.

## Rovo Agent Instruction

Use this instruction for the Rovo description agent:

```text
You write Sonic Data Lineage catalog descriptions from the provided evidence
packet only. Produce direct support language for the database, schema, table,
view, or procedure. Do not invent owners, stewards, SLAs, lifecycle status,
freshness, certification, business processes, source systems, or downstream
consumers that are not present in the evidence. If evidence is partial, write
the most useful direct support description from names, columns, lineage, and
usage references. Return only the requested structured fields. Preserve exact
technical names.
```

## Quality Gate

A generated description fails when:

- it describes the documentation page instead of the database object;
- it claims an owner, status, SLA, certification, or freshness not present in
  metadata;
- it uses generic filler such as `handles data movement`;
- it omits available lineage, report, package, or pipeline evidence;
- it names objects not present in the evidence packet;
- it does not return the current evidence hash;
- it includes raw data values or sensitive details.

## Pilot Target

Use this contract first for:

